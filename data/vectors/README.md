# Concept Vector Storage Format

This directory stores per-model concept vectors produced by
`concept-vectors/compute_concept_vectors.py`.

## Directory layout

Each model gets its own subdirectory:

```text
data/vectors/
  <model_output_dir>/
    baseline_mean.pt
    baseline_metadata.json
    <concept_slug>_all_layers.pt
    <concept_slug>_metadata.json
```

Current default output dir in this repo is:

```text
data/vectors/llama-3.3-70b-instruct/
```

## Filenames and slugs

- `concept_slug` is generated by lowercasing the concept and replacing any run
  of non `[a-z0-9]` characters with `_`, then trimming leading/trailing `_`.
- Example: `"Ice Cream!" -> "ice_cream"`.

## Tensor files (`.pt`)

All `.pt` files are written with `torch.save(...)` and currently store a single
`torch.Tensor` (not a dict) with shape:

```text
[num_layers, hidden_dim]
```

For the current `llama-3.3-70b-instruct` vectors in this repo, that is:

```text
shape = [80, 8192]
dtype = torch.bfloat16
```

Semantics:

- Row `i` is the vector for transformer layer `i` (0-based index).
- A single layer vector has shape `[hidden_dim]`.
- Concept vectors are computed as:
  `concept_vector = raw_sample_last_token_activations - baseline_mean`
  (elementwise, per layer).

### Files

- `baseline_mean.pt`
  - Mean of last-token activations across the baseline word list.
  - Shape `[num_layers, hidden_dim]`.
- `<concept_slug>_all_layers.pt`
  - Concept vector for that concept.
  - Shape `[num_layers, hidden_dim]`.

## Metadata JSON schema

### `baseline_metadata.json`

```json
{
  "model": "string",
  "num_words": "int",
  "words": ["string", "..."],
  "num_layers": "int",
  "hidden_dim": "int",
  "shape": [80, 8192],
  "hash": "16-char hex string",
  "timestamp": "ISO-8601 UTC string"
}
```

### `<concept_slug>_metadata.json`

```json
{
  "concept": "string",
  "slug": "string",
  "model": "string",
  "shape": [80, 8192],
  "hash": "16-char hex string",
  "raw_sample_hash": "16-char hex string",
  "baseline_hash": "16-char hex string",
  "timestamp": "ISO-8601 UTC string"
}
```

## Hash definition

`hash`, `raw_sample_hash`, and `baseline_hash` are the first 16 hex chars of
SHA-256 over:

1. tensor dtype string
2. tensor shape tuple string
3. raw contiguous tensor bytes

This is used for reproducibility/integrity checks.

## Consumer expectations

`prefill-detection/run_prefill_detection.py` loads vectors from:

```text
<vector_dir>/<concept_slug>_all_layers.pt
```

Then selects one layer by index and casts it to `torch.bfloat16`.

The loader also supports a legacy dict format (`data[layer_idx]`), but the
current writer in this repo emits plain tensors.
